# Rapid Invasion - Full Solve Writeup

## Challenge
- Name: `Rapid Invasion`
- Category: Reverse Engineering
- Hint: `Can we really view opcode in the plain or not?`
- Flag format: `ctfzone{...}`

## Provided Artifact
`rapid.zip` containing one binary:
- `rapid_invasion` (Mach-O 64-bit arm64, stripped)

## 1) Triage

I extracted and inspected the file:

```bash
unzip -o rapid_invasion/rapid.zip -d rapid_invasion/extracted
file rapid_invasion/extracted/rapid_invasion
strings -n 6 rapid_invasion/extracted/rapid_invasion
```

Key clues from strings:
- Prompt: `Enter the access code`
- Decoys:
  - `flag{this_is_not_the_flag_nice_try}`
  - `ctfzone{nope_not_this_one_either}`
- Success text:
  - `ACCESS GRANTED`
  - `The invasion begins...`

So the real flag is generated/validated at runtime.

## 2) Static Reverse

Because it is macOS arm64 and cannot run natively here, I reversed statically with `r2`:

```bash
r2 -q -c "aaa;afl" rapid_invasion/extracted/rapid_invasion
```

Found main:
- `main @ 0x100003360`

Disassembly showed:
1. Input length must be exactly `0x1e` (30 chars).
2. A custom VM interpreter executes bytecode from stack buffer.
3. Opcode dispatch table is in `__TEXT.__const` at `0x100003c78`.
4. Bytecode bytes are generated from constant table around `0x100003ccd`.

## 3) VM Opcode Mapping

From dispatch table (opcode -> handler):
- `0x10` -> set register from immediate
- `0x11` -> load input[index] into register
- `0x20` -> xor register
- `0x21` -> add immediate
- `0x22` -> sub immediate
- `0x23` -> rotate-left by immediate
- `0x30` -> compare register != immediate (sets fail temp)
- `0x31` -> assert previous compare was equal (otherwise fail)
- (others exist: stack and/or arithmetic ops)

## 4) What the Generated Program Does

For each of 30 input positions `i`, the generated bytecode enforces:

```text
r0 = input[i]
r0 ^= c1
r0 = (r0 + c2) & 0xff
r0 = (r0 - c3) & 0xff
r0 = rol8(r0, c4 & 7)
assert r0 == c5
```

Constants `c1..c5` come from the embedded table (`0x3ccd`) in 5-byte groups with a fixed offset scheme.

So each character is independent and can be solved directly by inversion:

```text
x  = ror8(c5, c4 & 7)
x  = (x + c3) & 0xff
x  = (x - c2) & 0xff
ch = x ^ c1
```

## 5) Recovered Flag

Applying this inversion for all 30 positions gives:

`ctfzone{r4p1d_vm_cr4ck3r_2026}`

I added an automated solver:
- `rapid_invasion/solve_rapid_invasion.py`

Run:

```bash
python3 rapid_invasion/solve_rapid_invasion.py
```

Expected output:

```text

```
